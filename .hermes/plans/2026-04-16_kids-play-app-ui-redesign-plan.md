# Kids Play App UI Redesign Plan

> 목표: 기능 프로토타입 느낌을 버리고, 27개월 아이가 혼자 눌러도 즐겁고 부모가 보기에도 완성도 있게 느껴지는 유아용 학습 앱으로 리디자인한다.

## 현재 문제 진단

현재 화면들은 공통적으로 다음 문제가 있다.

1. Generic Material 느낌이 너무 강하다.
   - 큰 파스텔 박스 + 기본 FilledButton 조합이 반복된다.
   - 앱만의 세계관이 보이지 않는다.

2. 시각 계층이 약하다.
   - 제목, 보조문구, 행동 버튼, 카드가 거의 같은 리듬으로 쌓인다.
   - 어디를 먼저 봐야 하는지 강한 유도가 없다.

3. 유아용 인터랙션 감성이 부족하다.
   - 눌렀을 때 즐거운 피드백, 캐릭터성, 놀이감, 공간감이 거의 없다.
   - 학습 앱이라기보다 내부 테스트용 관리자 화면처럼 느껴질 수 있다.

4. 화면 밀도 조절이 서툴다.
   - landscape의 낮은 height를 충분히 고려하지 못했다.
   - 큰 텍스트와 큰 카드가 동시에 들어가며 화면이 답답해진다.

5. 브랜드 일관성이 없다.
   - hero / home / category / learn / quiz가 하나의 디자인 시스템처럼 이어지지 않는다.

## 리디자인 방향 (기본안)

사용자 선택이 없으면 아래 기본안으로 진행한다.

- 방향: 프리미엄 유아 앱 + 자동차 놀이방 세계관 혼합
- 키워드: 둥근 형태, 깊이감 있는 레이어, 그림책 같은 배경, 장난감 같은 버튼, 또렷한 정보 계층
- 피해야 할 것:
  - generic material demo 느낌
  - 과한 텍스트 블록
  - 같은 크기의 박스 나열
  - 지나치게 비즈니스 앱 같은 정렬감

## 새 디자인 시스템 초안

### 색
- Primary Blue: 자동차/주행/행동 버튼
- Warm Yellow: 학습 카드 배경
- Coral Pink: 강조 텍스트, 축하/보상 포인트
- Mint Green: 성공/게임/보조 액션
- Cream/Cloud: 배경 레이어
- Navy: 본문/타이틀 텍스트

### 형태
- 단순 사각형 카드 금지
- 큰 radius + 살짝 두꺼운 내부 여백
- 그림자/하이라이트로 장난감 버튼 느낌 강화
- 카드 위에 아이콘/얼굴/스티커 요소를 겹쳐서 단조로움 제거

### 타이포그래피 원칙
- 제목 / 핵심 액션 / 문제 텍스트 / 보조 설명을 명확히 4단계로 분리
- 긴 설명문은 최대한 줄이고 1~2줄 내로 제한
- 학습 대상 문자(ㄱ, A, 3)는 항상 압도적으로 크게 보이게 함
- 문제 문구는 '이름'과 '찾아야 할 글자'를 분리 표기

### 인터랙션 원칙
- 눌러야 할 대상은 항상 1차 포커스를 받게 함
- 버튼은 평면이 아니라 '눌러보고 싶은 장난감'처럼 보이게 구성
- 성공 시 즉각적인 축하 모션/반응 여지 확보
- 비활성 버튼도 '고장난 버튼'처럼 보이지 않고 '곧 열릴 공간'처럼 보여야 함

## 화면별 개편 우선순위

### 1. HeroScreen
목표:
- 첫 인상이 지금보다 5배는 더 '앱 같다'고 느껴지게 바꾼다.

개편 포인트:
- 단순 gradient 배경 대신 하늘/도로/구름/자동차 놀이터 느낌 레이어 배경
- 얼굴 PNG를 그냥 중앙 배치하지 말고 '대표 캐릭터 배지'처럼 연출
- 제목 아래 설명문 축소
- 플레이 버튼은 하단 고정 대형 CTA 유지하되 더 입체적으로

### 2. HomeScreen
목표:
- '메뉴 3개 나열'이 아니라 '놀이터 3개 중 선택'처럼 보여야 함

개편 포인트:
- 한글/알파벳/숫자를 동일 박스 3개 대신 각각 다른 개성과 아이콘 부여
- 카드 안에 캐릭터/심볼/배경 패턴 추가
- 선택 시 확대/강조되는 구조 고려

### 3. CategoryHubScreen
목표:
- 학습하기 / 게임하기가 기능 버튼이 아니라 놀이 입구처럼 보여야 함

개편 포인트:
- 좌우 버튼을 동일한 일반 카드로 두지 않음
- 메인 액션 하나를 더 크게, 보조 액션 하나를 약간 작게 처리 가능
- 비활성 상태는 '곧 열려요' 느낌으로 처리

### 4. HangulLearnScreen
목표:
- 큰 문자 카드 자체가 예쁘고 집중감 있게 보이도록 개선

개편 포인트:
- 문자 카드와 설명 영역을 분리
- '문자' / '이름' / '힌트' 시각 계층 강화
- 카드 배경에 subtle texture 또는 sticker frame 도입
- 다음 버튼보다 카드가 주인공이 되게 조정

### 5. HangulQuizScreen
목표:
- 문제 카드 + 4지선다를 가장 완성도 높게 보이게 만들 것

개편 포인트:
- landscape height에 최적화된 2x2 정답 그리드 유지
- 문제 헤더와 선택지 간 간격 정리
- 선택지는 누르면 튀는 장난감 버튼처럼 보이게 개선
- 정답/오답 상태색, 축하 피드백 들어갈 자리 확보

## 구현 순서

1. 공통 디자인 토큰 정리
   - spacing
   - radius
   - shadow
   - color
   - text styles

2. 공통 컴포넌트 추출
   - primary CTA button
   - toy card
   - section header
   - quiz choice tile
   - screen background shell

3. Quiz 화면 먼저 리디자인
   - 이유: 사용자가 가장 바로 체감함
   - landscape 제약도 가장 엄격함

4. Hero → Home → CategoryHub 순으로 리디자인

5. Learn 화면 리디자인

## 완료 기준

다음 빌드에서 최소한 아래를 만족해야 한다.

- 사용자가 '프로토타입 같다'고 느끼지 않는다.
- 한 화면에서 가장 중요한 요소가 즉시 눈에 들어온다.
- 아이가 눌러보고 싶게 생긴 버튼/카드가 보인다.
- screenshot만 봐도 동일 앱의 화면이라는 일관성이 있다.
- landscape phone에서도 답답하거나 잘려 보이지 않는다.
