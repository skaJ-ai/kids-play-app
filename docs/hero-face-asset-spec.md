# Hero face asset spec for 승원이의 빵빵 놀이터

상태: input asset reviewed
기준 시각: 2026-04-16 16:14 KST

## 리뷰 대상
사용자가 전달한 웃는 얼굴 컷아웃 이미지.
목표 용도는 다음 두 가지다.
- 히어로 인트로의 얼굴 레이어
- 홈 화면 대표 캐릭터의 얼굴 레이어

## 현재 판단

이 이미지는 컨셉 적합성은 높다.
- 표정이 밝음
- 정면 구도에 가까움
- 친근함이 강함
- 스티커 같은 흰 외곽선이 있어 캐릭터성이 좋음

하지만 현재 파일은 프로덕션용 최종 자산으로 바로 쓰기에는 부족할 가능성이 높다.
가장 큰 이유는 체크보드가 실제 투명 배경이 아니라 이미지에 박힌 스크린샷/편집본일 가능성이 크기 때문이다.

즉, 이 파일은 아래 용도로는 좋다.
- 시안용
- 홈/히어로 레이아웃 검토용
- placeholder 대체 참고용

하지만 최종 앱 자산으로는 아래 조건의 재추출본이 필요하다.

## 권장 최종 자산 형태

### 1. 필수 최종본
- `hero_face_clean.png`
  - 얼굴만
  - 실제 투명 배경(alpha)
  - 외곽선/그림자 없음 또는 최소화

### 2. 권장 파생본
- `hero_face_sticker.png`
  - 얼굴 + 흰 외곽선
  - 실제 투명 배경
- `hero_face_sticker_shadow.png`
  - 얼굴 + 흰 외곽선 + 약한 그림자
  - 실제 투명 배경

## 권장 크기
- 512px
- 1024px
- 2048px

실무적으로는 1024px 또는 2048px 원본 하나만 있어도 1차 앱 구현에는 충분하다.

## 현재 이미지에서 좋은 점

1. 웃는 표정이라 첫인상이 좋다
2. 얼굴 구도가 안정적이라 작은 썸네일에도 잘 보일 가능성이 높다
3. 흰 외곽선이 있어 파스텔 배경/도로 배경 위에서 잘 뜬다
4. 홈 대표 캐릭터용으로 적합하다
5. 히어로 인트로에서 자동차와 같이 배치했을 때 시선 집중 효과가 좋다

## 현재 이미지에서 보정이 필요한 점

1. 체크보드 배경이 박혀 있을 가능성
- 반드시 실제 alpha 채널 포함 원본으로 다시 확보 필요

2. 외곽선이 다소 두껍게 느껴질 수 있음
- 히어로용은 조금 얇은 버전이 더 자연스러울 수 있음

3. 그림자가 baked 상태일 수 있음
- 앱에서 동적으로 shadow를 주는 쪽이 더 유연함

4. 해상도가 히어로 확대용으로는 다소 부족할 수 있음
- 원본 큰 사이즈 확보 권장

5. 머리카락 경계가 완벽한 누끼처럼 보이진 않음
- 최종 빌드 전 보정 권장

## 앱 내 권장 사용 방식

### 히어로 인트로
- 배경/자동차/로고와 분리된 얼굴 레이어로 사용
- 3~4초 인트로에서 차가 지나간 후 얼굴이 붙은 대표 캐릭터 또는 홈 대표 UI로 연결

### 홈 화면
- 앱 제목 근처 대표 캐릭터
- 또는 작은 원형/스티커형 프로필 배지로 사용

## 비권장 사용 방식
- 너무 크게 확대해서 전체 화면 절반 이상 차지하게 쓰는 것
- 체크보드가 박힌 상태로 그대로 번들하는 것
- JPG로 다시 저장해서 가장자리 투명도를 잃는 것

## 권장 파일 포맷

1순위:
- PNG RGBA

2순위:
- WebP lossless with alpha

비권장:
- JPG
- 체크보드가 포함된 스크린샷본

## 전달받으면 좋은 최종 자료

가장 좋은 것은 아래 셋 중 하나다.

A. 진짜 투명 배경 PNG 원본
B. 배경 제거 전 원본 사진 + 배경제거 요청
C. 편집 가능한 원본 파일(예: PSD/캔버스 export)

## 이 프로젝트에서의 실제 적용 결론

현재 받은 이미지는 hero face 컨셉 reference로 매우 좋다.
다만 최종 앱 자산으로는 아래 순서가 맞다.

1. 현재 이미지는 디자인 reference로 사용
2. 실제 투명 배경 hero face 최종본을 별도로 확보
3. 최종본은 git에 직접 넣지 않고 `assets/local_private/images/hero/hero_face.png` 또는 대응 WebP로 관리
4. build 시 `assets/generated`로 병합
